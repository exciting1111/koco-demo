# KOCO Demo - 前端静态页面开发说明

## 1. 项目简介

KOCO Demo 是一个为东南亚市场设计的高端电商营销活动（心愿购）前端静态演示页面。该页面采用纯前端技术栈（HTML5 + CSS3 + Vanilla JavaScript）构建，无需后端服务器即可在浏览器中直接运行和演示。

**核心功能特性：**
- **多语言支持**：内置越南语（默认）、简体中文、印尼语三种语言的实时切换。
- **响应式设计**：完美适配桌面端（PC）和移动端（手机），采用 Mobile-First 理念。
- **心愿购（Wish Purchase）**：商品分类筛选、进度条展示、动态中奖概率计算。
- **KOC 招募展示**：高端视觉卡片设计，展示 KOC 社区数据和福利。
- **交互式弹窗**：包含商品详情参与弹窗和"我的参与记录"历史弹窗。
- **登录状态模拟**：支持模拟未登录/已登录状态，控制特定 UI 元素（如参与记录按钮）的显示。

## 3. 技术栈与架构

- **HTML**：语义化标签，使用 `data-i18n` 属性标记需要翻译的文本节点。
- **CSS**：
  - 使用 CSS Variables（自定义属性）管理主题色（如 `--primary-color: #ff6b35`）。
  - 采用 Flexbox 和 CSS Grid 布局。
  - 包含平滑的过渡动画（`transition`）和悬浮效果（`hover`）。
  - 媒体查询（`@media (max-width: 768px)`）处理移动端适配。
- **JavaScript**：
  - 纯原生 JS（Vanilla JS），无任何第三方框架依赖（如 React/Vue/jQuery）。
  - 包含完整的翻译字典对象（`translations`）。
  - 负责 DOM 操作、事件监听、数据计算（如中奖概率）和状态管理。

## 4. 核心模块说明

### 4.1 多语言系统 (i18n)
在 `script.js` 中定义了 `translations` 对象，包含 `vn`（越南语）、`cn`（中文）、`id`（印尼语）三个子对象。
切换语言时，JS 会遍历所有带有 `data-i18n` 属性的 DOM 元素，并根据当前语言替换其 `textContent` 或 `placeholder`。

### 4.2 心愿购模块 (Wish Purchase)
- **数据结构**：商品数据硬编码在 `script.js` 的 `productData` 对象中，包含名称、价格、当前星数、总需星数等。
- **分类筛选**：点击分类标签（如"电子"、"美妆"）时，通过修改商品卡片的 `display` 属性实现实时过滤。
- **动态计算**：在商品详情弹窗中，用户选择投入的心愿星数量后，JS 会实时计算并更新：
  - 消耗的心愿星数量
  - 预期进度条（半透明橙色层）
  - 中奖概率（投入星数 / 总需星数 * 100%）

### 4.3 登录状态模拟
页面右上角有"登录"和"注册"按钮。点击后会触发 `showLoggedIn()` 函数，隐藏登录/注册按钮，显示用户头像、星星余额，并显示心愿购区块的"我的参与记录"按钮。

## 5. 如何运行与部署

### 5.1 本地运行
由于是纯静态文件，直接在浏览器中双击打开 `index.html` 即可运行。
如果需要测试某些受跨域限制的特性，建议使用本地服务器运行：
```bash
# 使用 Python 启动本地服务器
python3 -m http.server 8080
# 然后在浏览器访问 http://localhost:8080
```

### 5.2 部署到生产环境
该项目可以直接部署到任何静态文件托管服务，如：
- GitHub Pages
- Vercel
- Netlify
- AWS S3 / 阿里云 OSS
- Nginx / Nginx 静态目录

## 6. 后续开发建议（接入真实后端）

如果需要将此 Demo 转化为真实产品，建议进行以下改造：
1. **框架迁移**：将原生 HTML/JS 迁移到 React/Vue/Next.js 等现代前端框架，以便更好地管理组件状态。
2. **API 对接**：
   - 将 `productData` 替换为从后端 API 获取的真实商品列表。
   - 将"立即参与"按钮的点击事件绑定到真实的扣费/抽奖接口。
   - 接入真实的 JWT/OAuth 登录鉴权系统。
3. **防羊毛党机制**：在注册和参与流程中加入手机号 OTP 验证和设备指纹识别。
